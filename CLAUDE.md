# CLAUDE.md

## Project Overview

Kingdom Health (kristyfitness) is a faith-centered 90-day wellness coaching website with a blog/journal powered by Keystatic CMS. The site is built with Astro, deployed on Vercel with SSR, and uses GitHub-backed content management in production.

**Live site:** https://www.kingdomhealth.fitness

## Tech Stack

- **Framework:** Astro 5 (SSR via `@astrojs/vercel`)
- **UI:** React 19 for interactive components, Astro components for pages/layouts
- **CMS:** Keystatic (local mode in dev, GitHub mode in production)
- **Language:** TypeScript (strict mode)
- **Styling:** Vanilla CSS with custom properties (no CSS framework)
- **Deployment:** Vercel (server output mode)
- **Node:** 22 (see `.nvmrc`)

## Quick Start

```bash
npm install
npm run dev        # Start dev server at localhost:4321
npm run build      # Production build
npm run preview    # Preview production build
npm run check      # TypeScript & Astro validation
```

No environment variables are required for local development.

- `scripts/` — SEO automation (weekly checks, IndexNow submission)
- `.github/workflows/` — CI/CD (weekly SEO audit, IndexNow on push)

## Key Conventions

### Pages & Routing

- Static pages use `export const prerender = true` in their frontmatter
- Dynamic routes (e.g., `[slug].astro`) use `getStaticPaths()`
- The Keystatic admin routes (`/keystatic`) must NOT be prerendered

### Content Management (Keystatic)

Blog posts are stored in `src/content/blog/{slug}/` directories with YAML frontmatter and a Markdoc document body. The schema is defined in `keystatic.config.ts`.

**Blog post fields:** title, slug (kebab-case), publishDate, updatedDate, excerpt, coverImage, tags, author (default: "Kristy"), draft, seoTitle, seoDescription, body.

- In dev: content is read/written directly from the filesystem
- In production: content is managed via GitHub API (requires OAuth app setup)
- Draft posts (`draft: true`) are hidden from the live site

### Styling

All styles live in `src/styles/global.css` using CSS custom properties:

- `--bg: #efe6d8` (warm beige background)
- `--accent: #8a6a32` (gold/brown accent)
- `--radius: 22px` (border radius)
- Fonts: Cormorant Garamond (headings), Work Sans (body)
- Fluid typography and spacing via `clamp()`

No CSS framework or preprocessor is used. Add new styles to `global.css`.

### SEO

- JSON-LD schema markup is embedded in `BaseLayout.astro`
- Open Graph and Twitter Card meta tags on all pages
- RSS feed at `/rss.xml` via `@astrojs/rss`
- Sitemap auto-generated by `@astrojs/sitemap`
- `robots.txt` and `llms.txt` in `public/` for crawler discoverability
- IndexNow submission triggered on push to main (GitHub Action)

### Forms

The homepage contact form uses Formspree. It includes a honeypot field (`hp-field`) for spam prevention and hidden UTM tracking fields.

## Common Tasks

### Adding a blog post

Use the Keystatic editor at `localhost:4321/keystatic` during development, or create a directory under `src/content/blog/{slug}/` with the required YAML frontmatter matching the schema in `keystatic.config.ts`.

### Modifying site metadata

Edit `src/lib/site.ts` for site-wide constants (URL, tagline, social links). Page-specific SEO is set in each page's frontmatter or the `BaseLayout` component props.

### Updating styles

Edit `src/styles/global.css`. Use existing CSS custom properties for consistency. The design uses a warm, earthy color palette with soft shadows and rounded corners.

## CI/CD

- **IndexNow** (`.github/workflows/indexnow.yml`): Submits pages to search engines on push to main when src/, public/, or config files change
- **SEO Weekly** (`.github/workflows/seo-weekly.yml`): Runs Monday at 2 PM UTC, checks indexing status and sitemap health

## Important Notes

- Always run `npm run check` before committing to catch TypeScript and Astro errors
- Cover images for blog posts upload to `public/blog/covers/`
- The site targets a faith-centered, professional, warm tone — keep copy consistent with this voice
- Image loading: use `loading="eager"` for above-the-fold hero images, `loading="lazy"` for everything else
